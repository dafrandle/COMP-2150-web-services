Executive Summary

 XML is designed to store and transport data. However, XML does nothing. XML just uses tags to categorize data that is contained inside them. Because of that it should be quite self-descriptive. Because of the use of tags most applications using XML will work as expected even if tags or data is added or removed. XML is useful for designing data storage for business programs to use. (not databases those are for humans) It also allows you to design properties that programs can use and edit. Overall it allows businesses to store information for it's systems in a self described, standardized, easier to read format.

What an XML parser does.

An XML parser reads XML files and makes the data within accessible to applications and programing languages To do this it loads it into a XML DOM, or Document Object Model. The DOM defines the properties and methods for accessing and editing XML.
